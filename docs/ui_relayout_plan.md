# 前端布局回正计划 v3

## 当前问题诊断

1. **仿真展示区 (sim-area)**: 固定 180px 高，3 列 sim-card 信息密度极低，
   RViz 预留列只有一句话，视觉上是大面积空白。占据左侧约 25% 高度。
2. **右侧栏**: 280px 太窄，中文标签+数值对挤在一起，操作按钮也拥挤。
3. **底部信息区**: max-height:200px 且 overflow:hidden，4 张卡被严重压缩。
4. **地图区**: simple 模式只有一个 #0f172a 纯色矩形，看起来像空白/加载失败。
   没有网格线、没有坐标标注、没有港口轮廓——答辩时观感差。
5. **比例失衡**: sim-area 太高 + 底部太矮 = 地图被上下夹击，不是"视觉主角"。

## 修复方案

### 布局
```
grid-template-rows: 36px 1fr 210px 24px
grid-template-columns: 1fr 330px
```

- 左侧: sim strip (flex: 0 0 约 28-30%) + 地图 (flex: 1, 约 70%)
- sim strip: 单行横排摘要，不再三列空洞卡片
- 右侧: 330px 固定, 三卡 (状态/任务/控制)
- 底部: 210px, 4卡横排, 允许滚动

### 地图 (simple 模式视觉增强)
- 保留深色背景，但添加:
  - 默认显示 10m 间距浅色网格线
  - 添加 X/Y 坐标轴标签 (原点标记)
  - 添加港口设施轮廓线 (简化多边形: 码头、堆场、水域)
  - 添加"仿真平面坐标 (米)"提示
- OSM/Carto 模式保持不变，加 fallback

### 启动脚本
- start_all.sh = 主入口 (已确认)
- dev_start.sh = 调试入口
- README/demo.md 已更新
